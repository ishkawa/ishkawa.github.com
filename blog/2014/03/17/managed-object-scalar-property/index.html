NSManagedObjectのプリミティブ型プロパティ

iOS 4までのNSManagedObjectはプリミティブ型のアクセサを自動生成できなかったので、数値はNSNumberとして持つ必要がありました。 つまり、以下のようなプロパティを提供する場合には、アクセサを自分で実装する必要があったのです。

Core Data Programming GuideのNon-Standard Persistent Attributesによると、 NSNumberのプロパティを内部的に持ち、flagのアクセサを以下のように実装する必要があったようです。

こういったコードが増えるとミスの原因になるので嫌だなあと思っていたら、 iOS 5からはスカラについてはこのような実装は不要になったと教えてもらいました。

NSManagedObjectにプリミティブな型のプロパティをつけるにはこういう鈍い方法を取るしかないのかな https://t.co/8IORgL6gxt

— Yuichi Fujishige (@nakiwo) 2014, 3月 16

@_ishkawa WWDC2011-Session 303と https://t.co/8xzDL4uIVK から、iOS 5以後の機能のようです。(その他ドキュメントには見当たらず)

— Yuichi Fujishige (@nakiwo) 2014, 3月 17

xcdatamodelで定義する型とNSManagedObjectのプロパティの型の対応がちょっとわかりづらいのですが(例えばInteger 64はint64_t)、 XcodeでNSManagedObjectを作成する際に”Use scalar properties for primiteve data types”にチェックをつけておくと、 対応したプリミティブ型のプロパティが用意されます。 “Use scalar properties for primiteve data types”にチェックをつけておくとすべてプリミティブ型でプロパティが用意され、 チェックを外すとすべてNSNumberでプロパティが用意されますが、プリミティブ型のプロパティとNSNumberのプロパティを混在させることもできます。
